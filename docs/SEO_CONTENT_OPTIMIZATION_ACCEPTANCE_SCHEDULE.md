# HOKMeta SEO 内容优化与验收排期

更新日期：2026-07-17  
执行周期：2026-07-17 至 2026-08-14  
适用范围：英文、繁体中文、印尼语、菲律宾语；首页、英雄、克制、攻略、工具、趋势、版本、sitemap 与 GSC。

## 一、目标与原则

本排期执行 [网站整体优化方案](./HOKMETA_SITEWIDE_OPTIMIZATION_PLAN.md)，并受以下文档约束：

- [任务执行 SOP](./HOKMETA_TASK_EXECUTION_SOP.md)
- [内容与页面改动约束](./HOKMETA_CONTENT_AND_PAGE_GUARDRAILS.md)
- [多语言上线计划](./I18N_ROLLOUT_PLAN.md)

目标不是机械增加字数或批量制造页面，而是让每一个可收录页面同时具备：

1. 明确的玩家搜索意图。
2. 可验证的数据、来源与更新时间。
3. 合理的下一步路径：英雄、克制、攻略、工具、装备。
4. 当前 locale 的真实正文，而非英文 fallback。
5. 可由本地构建、导出 HTML 和线上预览验证的结果。

## 二、统一状态定义

| 状态 | 含义 | 是否出现在语言切换器 / sitemap / GSC |
|---|---|---|
| `已完成并可收录` | 页面内容、内链、metadata、当前语言正文和验证均通过 | 是 |
| `技术已接入` | 路由、数据或 UI 已存在，但正文或用户路径未完成 | 否 |
| `已生成未开放` | 构建可生成，用于内部检查或翻译校对 | 否 |
| `待人工内容` | 需要人工审核、资料补充或高质量翻译 | 否 |

以下情况一律不得标记为“已上线”：只出现语言按钮、只生成路径、只翻译 UI、正文英文 fallback、没有可见内链、没有验证导出 HTML。

## 三、验收标准

### 1. 所有可收录页面

- Title、H1、description 与页面搜索意图一致，不堆砌关键词。
- 主体内容在静态 HTML 中可见，不依赖浏览器请求后才显示。
- 有至少一个可见的上游入口和两个相关下游链接。
- canonical、hreflang、sitemap 只包含真实已开放的 locale。
- 数据页仅展示事实、状态、来源和下一步；不自动生成无证据因果解释。

### 2. 英雄详情与克制页

- 首屏有 build、Arcana、Spell、当前数据、更新时间和来源。
- 提供 Guide、Counter、Damage Calculator、Build Compare 与相关装备的可见链接。
- FAQ 仅覆盖真实搜索意图；FAQ Schema 与可见问题答案一致。
- 30 天趋势只显示当前值、7 日 / 30 日变化、状态和可引用的官方改动；没有证据则明确不推断原因。

### 3. Learn 攻略页

- 首段直接回答搜索问题，不用泛泛导语。
- 包含可操作的出装 / 对局 / 团战 / 替换场景，不只重复英雄数据。
- 有来源、作者、发布时间、更新时间、FAQ 和站内深链。
- 所有面向用户的标题、正文、小标题、FAQ、CTA、内链锚文本必须是当前 locale 的语言。

### 4. 工具与趋势页

- 静态 HTML 保留标题、用途、数据来源、Schema 和下一步深链。
- 客户端仅加载交互必要数据；不传递完整英雄或装备数据造成静态产物膨胀。
- 趋势页面只作事实型榜单；深度“为什么”另建人工审核的 Learn 内容。

### 5. 最小验证命令

```bat
npm run build
npm run qa-zh-locale
npm run gsc-priority-urls
```

按任务补充：导出 HTML 抽查、sitemap 检查、四语正文检查、预览环境链接和工具交互检查。

## 六、每周验收会议清单

每周验收按以下顺序进行，未通过不进入下一阶段：

1. **内容**：本周页面是否真正回答目标查询，是否存在重复、空话或无证据结论。
2. **链接**：Hero、Counter、Guide、Tool、Item 是否能互相到达；重点页是否孤岛。
3. **语言**：各 locale 的正文、FAQ、CTA、锚文本是否一致；是否有英文 fallback。
4. **抓取**：sitemap、canonical、hreflang、robots、GSC 优先 URL 是否匹配开放状态。
5. **技术**：构建、静态 HTML、核心交互、`out/` 体积与 Cloudflare 预览是否正常。
6. **数据**：Camp 同步和 D1 快照日期是否最新，趋势内容是否标注来源和限制。

## 八、变更控制

- 任一阶段发现要新增超过 3 个页面族、同时改数据源和页面结构、或需开放新 locale，必须重新输出任务评估卡并获得确认。
- 不以“导航出现”“页面能生成”“有语言按钮”作为完成证据。
- 不为了 SEO 分数补无价值字数，不为了新鲜度批量刷新更新时间，不为了外链复制发布同一篇推广文。
- 每次上线后的 GSC 观察窗口至少为 14 天；早期优先看曝光和进入前 30 的查询，不以单日点击判断内容是否有效。

## 九、最终验收目标（2026-08-14）

到 2026-08-14，HOKMeta 的最低可验收状态为：

1. 已废弃 Tier List 不再出现在 sitemap、GSC 提交清单和站内核心入口。
2. sitemap 的更新日期能反映实际页面或数据变动，而非全站统一刷新。
3. 首页、Hero Trends、Tools 和 Top 20 英雄形成清晰的站内路径。
4. 7 个已有 GSC 曝光的英雄完成英文/繁中 Hero + Counter + Guide + Tool 深链集群。
5. 趋势数据页保持客观；解释性文章只在人工审核的 Learn 页面发布。
6. ID/FIL 仅开放通过完整正文验收的页面，并有明确完成清单。
7. 每项上线均有构建、导出 HTML、链接和 locale 检查证据。
